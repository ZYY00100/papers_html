AI学习路线图：20个核心概念的全景指南

No PhD required. No jargon. Just simple explanations and visuals.

2026年你必须理解的20个人工智能概念

PART 1

人工智能的实际运作方式

一切构建的基础

01

Neural Networks · 神经网络

The brain of every AI model. 每个AI模型的大脑。

  • 神经网络是一条由多层组成的管道
  • 数据进入输入层 → 通过隐藏层 → 作为预测结果输出
  • 每个连接都有一个"权重"——控制一个神经元对下一个神经元影响程度的小分数
  • 训练 = 调整数十亿个权重，直到输出准确
GPT-4 拥有约 1.8 万亿个参数。Claude 3 Opus 拥有数千亿。所有这些都源自同一个基本概念：分层神经元，连接可调。

02

Tokenization · 分词化

在AI读取你的文本之前，它会把它拆分成称为标记的部分。

  • 不总是完整的词汇："playing" → "play" + "ing"
  • "ChatGPT" → "Chat" + "G" + "PT"
  • 语言很混乱：新词、拼写错误、混合语言，固定词汇量将极其庞大
  • 即使模型从未见过某个单词，也能通过将其拆成熟悉的部分来理解
粗略规则：1 token ≈ 0.75 个单词。1000 tokens ≈ 750 个单词。

03

Embeddings · 嵌入

每个 token 变成一个数字——一个代表含义的向量。就像词语的 Google Maps。

  • "Doctor" 和 "Nurse" 在向量空间中位置接近
  • "Doctor" 和 "Pizza" 在向量空间中位置遥远
  • "King" - "Man" + "Woman" ≈ "Queen"
  • 模型不按你理解的方式理解词语——它理解的是距离和方向
这就是语义搜索、推荐系统、RAG 系统背后的核心技术。所有"理解意图"的东西底层都用嵌入。

04

Attention · 注意力机制

"Apple" 在不同语境中含义不同——注意力让模型知道该关注什么。

  • "I ate an Apple" → 水果
  • "I bought Apple stock" → 公司
  • 嵌入本身无法解决这个问题，但注意力可以
  • 注意力让每个词查看句子中的所有其他词，决定什么重要
在注意力机制之前，模型从左到右阅读，速度慢且受限。之后，模型同时看到整个句子。这个单独的想法开启了现代 AI。

05

Transformers · 变换器

驱动几乎所有 AI 模型的架构。2017年在论文《Attention Is All You Need》中引入。

  • 突破：不用逐字阅读，而是用注意力并行处理所有内容
  • 工作流程：文本 → 分词 → 嵌入 → 堆叠注意力层 → 输出
  • 每层逐步精炼理解：早期层看语法和基本结构，中期层看词语关系，深层做复杂推理
  • GPT、Claude、Gemini、Llama、Mistral——都是变换器
如果你理解这一个架构，你就理解了现代 AI。

PART 2

LLM 如何工作

当你与 AI 对话时，实际发生了什么

06

LLMs · 大语言模型

LLM 是一个在大量文本上训练过的变换器。书籍、网站、代码、维基百科、Reddit，数万亿个 token。

  • 训练任务听起来简单到不可思议：预测下一个 token
  • 但当你在数万亿个例子上重复这个操作，令人惊叹的事情发生了
  • 模型学会了语法，然后是推理，然后是写代码、翻译语言、解数学题
  • 没有人告诉它要做这些——它从大规模 next-token 预测中涌现出来
"Large" = 数千亿个参数。训练成本 = 数百万美元。ChatGPT、Claude、Gemini 都是 LLM。

07

Context Window · 上下文窗口

每个 AI 模型都有记忆极限，叫做上下文窗口——模型一次能"看到"的最大 token 数。

  • 早期 GPT：约 4,000 tokens。GPT-4：128,000 tokens。Claude 3.5：200,000 tokens。Gemini 1.5 Pro：1,000,000 tokens
  • 更大的窗口 = 更多的上下文 = 更好的答案
  • 但有个问题：模型不会均匀读取所有内容，它们聚焦在开头和结尾
  • 中间部分？经常被忽略——这叫做"中间迷失"问题
大上下文窗口 ≠ 完美的记忆。理解这个解释了为什么 AI 有时"忘记"你明明提到过的东西。

08

Temperature · 温度

AI 生成文本时有一个叫做温度的旋钮，控制创意的程度。

  • Temperature = 0：总是选择最安全、最可预测的词
  • Temperature = 1：更有创意地选择，更多样化
  • Temperature = 2+：变得狂野，有时不连贯
低温度 → 适用于代码、事实、摘要。高温度 → 适用于头脑风暴、创意写作、变体生成。理解这个解释了为什么 AI 有时看起来"无聊"，有时又让你惊喜。

09

Hallucination · 幻觉

AI 自信地撒谎。不是故意的——它真的无法避免。

  • LLM 不搜索真理——它预测最可能出现的下一个 token
  • 如果一个虚假陈述基于训练模式看起来"应该接下来出现"，它就会生成它
  • 没有验证，没有查找，纯模式匹配
  • 所以它会：引用一个不存在的论文，发明一个从未创建过的 API 函数，以完全自信的态度陈述一个虚假的历史"事实"
修复方法：永远不要在未验证的情况下相信 AI 输出的事实。使用 RAG（概念 16）让它基于真实数据回答。

10

Prompt Engineering · 提示工程

你提问的方式决定一切。同样的模型，同样的问题，框架不同结果截然不同。

  • 糟糕的提示："解释 API" → 得到模糊、表面化的答案
  • 好的提示："解释 REST API 如何处理身份验证，给出一个带代码的真实例子，假设我是初级开发者" → 得到具体、结构化、直接有用的答案
  • 真正有效的技巧：给背景、分配角色、展示例子、明确输出格式、把复杂请求分解成步骤
提示工程不是 hack。它是你与模型沟通的主要方式。

PART 3

AI 模型如何改进

从原始模型到可用产品

11

Transfer Learning · 迁移学习

从头训练是昂贵的。迁移学习解决了这个问题。

  • 你用一个已经在大型通用任务上训练过的模型，然后为特定任务适配它
  • 你不是从零开始——而是在其基础上构建
  • 就像：你已经知道如何骑自行车 → 学习摩托车会快得多，因为你已经转移了已有的知识
  • 这就是今天几乎所有 AI 产品的运作方式
OpenAI 训练大型基础模型 → 公司为其特定用例微调 → 节省数百万美元的计算成本和数月的训练时间。没有公司再从零开始训练了。

12

Fine-Tuning · 微调

迁移学习告诉你概念，微调告诉你如何做。

  • 你拿一个预训练模型，在一个更小、更专注的数据集上继续训练
  • 模型已经会说"语言"了，现在你在教它你的特定领域
  • 例子：医学模型在临床笔记上微调，法律模型在合同上微调，编码模型在 GitHub 上微调
  • 结果：一个为你的用例完美响应的模型
成本：你需要更新数十亿个参数，需要多 GPU 和强大基础设施。这就是 LoRA（下一个概念）如此重要的原因。

13

RLHF · 人类反馈强化学习

微调让模型专业化。RLHF 让它们感觉有用且安全。

  • 没有 RLHF：模型只是预测文本，流畅但不对齐
  • 有了 RLHF：模型学习人类实际喜欢什么
  • 工作原理：展示提示 → 模型生成多个响应 → 人类对响应排序 → 模型学习偏好什么
  • 重复数千次，模型建立"好答案"的感觉：清晰、有帮助、诚实、安全
这就是 ChatGPT 和 Claude 感觉像助手而不是随机文本生成器的原因。没有 RLHF，它们仍然会很出色，但用处小得多，不那么值得信任，也更难控制。

14

LoRA · 低秩适应

微调很强大但昂贵。LoRA 解决了这个问题。

  • 不改变整个模型，而是：保持原始模型冻结，在顶部添加微小的可训练层
  • 这些层只是完整模型大小的一小部分
  • 洞察：大多数微调改变很小，你不需要重写整个模型，只需要小的针对性调整
  • 结果：在单张消费级 GPU 上微调成为可能，存储一个基础模型 + 交换不同的 LoRA 适配器变得实用
LoRA 是开源 AI 爆发的原因。突然任何人都可以在笔记本电脑上微调强大的模型。

15

Quantization · 量化

模型越来越大，运行它们需要大量内存和计算。量化让它们更小更便宜。

  • 方法：降低每个权重的精度
  • 以全精度存储的权重使用 32 位，量化到 4 位 → 小 8 倍
  • 疯狂的是：质量下降通常很小
  • 这就是为什么你现在可以：在 MacBook 上运行 LLaMA，在消费级 GPU 上本地运行 Mistral，在手机上运行强大模型
没有量化，大型模型将保持在数据中心。有了量化，它们可以在你的机器上运行。

PART 4

真实 AI 系统如何构建

你实际使用的产品背后的逻辑

16

RAG · 检索增强生成

LLM 产生幻觉是因为它们从记忆回答。RAG 通过让它们先查找来修复这个问题。

  • 用户提问
  • 系统在知识库中搜索相关文档
  • 将这些文档作为上下文传递给模型
  • 模型使用真实信息回答——不是猜测
闭卷考试（无 RAG）：从记忆回答，经常错误。开卷考试（RAG）：检查来源，准确得多。每个严肃的 AI 产品都使用 RAG：客户支持机器人、法律工具、医学助手、内部知识库。

17

Vector Databases · 向量数据库

RAG 需要快速找到正确的文档。但如何通过含义而不仅仅是关键词搜索数百万份文档？

  • 每个文档转换为嵌入（数字向量）
  • 这些向量存储在数据库中
  • 当用户提问时，问题也变成向量
  • 数据库找到与问题向量最接近的向量，返回语义最相似的文档
"heart disease treatment" 能找到关于 "cardiac care protocols" 的文档，即使词语不完全匹配。工具：Pinecone、Qdrant、Weaviate、pgvector。向量数据库是 AI 系统"理解"而不仅仅是"匹配字符串"的原因。

18

AI Agents · AI 代理

LLM 回应消息。AI 代理实际做事。

  • 区别：LLM = 你问，它答，结束。Agent = 你给目标，它计划、行动、检查结果、调整、重复
  • 代理循环：思考 → 行动 → 观察 → 重复
  • 例子：编码代理修复 bug → 阅读问题 → 探索代码库 → 识别问题 → 写修复 → 运行测试 → 看看哪里失败 → 调整修复 → 重复直到完成
  • 模型是大脑，工具是手
代理可以使用什么工具？Web 搜索、代码执行、文件系统、API、邮件/日历、数据库。代理是将 AI 从聊天机器人变成同事的原因。

19

Chain of Thought · 思维链

有时 AI 答错不是因为它笨，而是因为它太快得出答案。

  • 不直接问最终答案："解决：如果火车以 60 mph 行驶 2.5 小时，多远？"
  • 让它逐步思考："逐步解决：速度 = 60 mph，时间 = 2.5 小时，距离 = 速度 × 时间 = ？"
  • 模型逐步推理：第 1 步：确定公式 → 第 2 步：代入数字 → 第 3 步：计算
  • 对于数学、逻辑、多步骤问题可靠得多
洞察：给模型思考的空间，而不仅仅是反应。这就是为什么"逐步思考"或"仔细推理"的提示实际上有效。

20

Diffusion Models · 扩散模型

到目前为止都是关于文本的。扩散模型解释 AI 如何生成图像。这个过程是反直觉的。

  • 模型不学习画画——它学习销毁图像
  • 训练：从真实图像开始，逐步添加噪声直到纯静态，训练模型反转这个过程——逐步去除噪声
  • 生成：从纯噪声开始，模型逐步去除噪声，根据你的文本提示引导，图像从随机中浮现
  • 这个名字来自物理学——粒子随机扩散通过介质，就像墨水在水中扩散
不再只是图像：视频（Sora、Runway）、音频、3D 内容、药物分子。扩散模型是 AI 生成任何视觉内容的方式。
