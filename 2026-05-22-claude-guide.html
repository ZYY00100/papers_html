PART 01

从打开它的第一秒就做对

01 建一个 Project，别每次都从零开始

每次新开一个 Chat，Claude 都是空的。它不认识你，不了解你的工作，不知道你的目标，也不清楚你喜欢怎么沟通。

于是你花前几条消息重新解释自己——或者干脆不解释，然后 Claude 给你的东西泛泛的，怎么都不太对。

Projects 解决这个问题。一个 Project 是一个持久的工作空间，Claude 会记住其中每一次对话的上下文。建一次，之后每次进来它都已经认识你了。

去 Claude，侧边栏点 Projects，建一个新的。起个名字，比如"工作"或者"日常"。之后所有对话都放在这个 Project 里。

02 告诉 Claude 你是谁

大多数人在这一步之前就开始问了，然后奇怪为什么 Claude 给的答案总觉得哪里不对。

把你的背景一次性存进 Project 里，Claude 会在每次对话开始前自动读取。格式不重要，诚实填完每一项最重要。

复制以下模板，填好后存入 Project：

Prompt 
我的名字是 [你的名字]。

我从事 [你的职业/角色]。日常工作主要包括 [2-3 件你实际在做的事]。

目前我最关注的目标是 [1-3 个你正在努力的具体目标]。

我主要用 Claude 来做 [主要使用场景——写作、研究、分析、学习、编程等]。

我的背景和知识水平：[哪些领域熟悉，哪些正在学，哪些完全陌生]。

我喜欢的信息接收方式：[例如：直接简洁 / 详细带例子 / 一步一步来 / 不要列表 / 段落短一点]。

我不想要的：[例如：不要加免责声明、不要用企业腔、不要重复我说的话、不要开头说"好问题"]。

我关注的话题和领域：[你的兴趣、行业、细分方向]。

Prompt

我的名字是 [你的名字]。我从事 [你的职业/角色]。日常工作主要包括 [2-3 件你实际在做的事]。目前我最关注的目标是 [1-3 个你正在努力的具体目标]。我主要用 Claude 来做 [主要使用场景——写作、研究、分析、学习、编程等]。我的背景和知识水平：[哪些领域熟悉，哪些正在学，哪些完全陌生]。我喜欢的信息接收方式：[例如：直接简洁 / 详细带例子 / 一步一步来 / 不要列表 / 段落短一点]。我不想要的：[例如：不要加免责声明、不要用企业腔、不要重复我说的话、不要开头说"好问题"]。我关注的话题和领域：[你的兴趣、行业、细分方向]。
03 把这些变成 Custom Instructions

把你的背景存进去是第一步。Custom Instructions 更进一步，不只告诉 Claude 你是谁，还告诉它和你打交道时默认应该怎么做。

把上面那段填好后，再发给 Claude 这段话：

Prompt 
根据你了解的关于我的一切，为这个 Claude Project 写一套自定义指令。

指令需要做到：
- 描述我是谁、做什么工作
- 设定我默认的沟通风格和格式
- 告诉 Claude 和我工作时绝对不要做什么
- 定义我希望在每个回复中看到的语气
- 包含我希望在任何对话中默认启用的行为

用第二人称写，就像 Claude 在读帮助我时需要遵守的规则一样。要具体，不要泛泛的建议。控制在 400 字以内。

Prompt

根据你了解的关于我的一切，为这个 Claude Project 写一套自定义指令。指令需要做到：描述我是谁、做什么工作；设定我默认的沟通风格和格式；告诉 Claude 和我工作时绝对不要做什么；定义我希望在每个回复中看到的语气；包含我希望在任何对话中默认启用的行为。用第二人称写。要具体，不要泛泛的建议。控制在 400 字以内。

把输出结果复制粘贴到 Project Instructions 里。这会成为这个 Project 下所有对话的永久运行模式。

PART 02

先改变对 Claude 的认知

04 Claude 不是搜索引擎

大多数人用 Claude 的方式和用 Google 一样：打一个问题，等答案。这是最低价值的用法。

Claude 不是检索工具。它是一个思考伙伴。它不只是调取信息，而是推理、综合、质疑、在上下文基础上构建。把它当搜索引擎用，实用价值损失 80%。

不要再问 Claude 某样东西是什么。开始让它帮你思考某件事。

不要问："什么是 prompt caching？"

试着问："我在做一个工作流，每轮对话要调 Claude 20 次。帮我理清 prompt caching 的原理，以及它在我这种情况下是否真的能降低成本。"

第二个提问方式给 Claude 一个需要和你一起解决的问题。第一个只是让它背一段定义。

05 让 Claude 先问你问题

这是最强的方法之一，但几乎没人用。在 Claude 开始做任何复杂任务之前，告诉它先向你收集信息。

Claude 在开始前先提问，最终输出质量会明显更好，因为它是建立在正确的地基上的。否则 Claude 只能靠猜，然后你花时间纠正那些本来可以一步到位的内容。

任何重要任务之前用这段：

Prompt 
在开始之前，先问我 5 个最重要的问题，帮助你把这件事做好。等我回答之后，再开始。

Prompt

在开始之前，先问我 5 个最重要的问题，帮助你把这件事做好。等我回答之后，再开始。

或者针对具体任务：

Prompt 
我需要你帮我给潜在客户写一封开发信。在你写任何内容之前，先问我需要知道什么，才能让这封信真正好，而不是泛泛的。

Prompt

我需要你帮我给潜在客户写一封开发信。在你写任何内容之前，先问我需要知道什么，才能让这封信真正好，而不是泛泛的。

PART 03

用得比大多数人都深

06 风格克隆

当 Claude 在没有示例的情况下替你写作，它会用它自己的风格写。输出语法正确，但语气完全不对——听起来就像 AI 写的东西。

给 Claude 三到五篇你自己的文章。让它分析你的规律，而不只是描述你的风格。之后它写出来的东西就像你写的，而不是像一个 polished 的企业助手。

Prompt 
以下是我写的 3 篇示例：

[粘贴示例 1]
[粘贴示例 2]
[粘贴示例 3]

详细分析我的写作风格。注意：句子的长度、节奏、词汇选择、如何开头和结尾段落、我在避免什么、我的语气是正式还是随意、以及任何让我的写作与众不同的地方。

之后当我要求你为我写任何东西时，严格匹配这个风格。不要用你自己的默认模式。

Prompt

以下是我写的 3 篇示例：[粘贴示例 1][粘贴示例 2][粘贴示例 3] 详细分析我的写作风格。注意：句子的长度、节奏、词汇选择、如何开头和结尾段落、我在避免什么、我的语气是正式还是随意、以及任何让我的写作与众不同的地方。之后当我要求你为我写任何东西时，严格匹配这个风格。不要用你自己的默认模式。
07 Claude 作为你的对抗性思考伙伴

大多数人用 Claude 帮忙出主意。Claude 在你说的基础上叠加、增加、扩展。你得到的是认同和补充。

这有时候有用，但不是压力测试一个想法的方式。

在任何计划、决定、或者文章定稿之前，让 Claude 来攻击它。不是点评，是攻击。区别很重要。

Prompt 
这是我的计划：[描述你的计划]

你的任务是摧毁它。找出我每一个可能错误的假设。找出这件事可能失败的所有方式。尽可能强硬地为相反观点辩护。不要客气。不要加限定条件。只管攻击。

之后，为我的立场做最强的论证。建构一个你能构建的最有力的理由来说明为什么我是对的。

然后告诉我你真实的想法。

Prompt

这是我的计划：[描述你的计划] 你的任务是摧毁它。找出我每一个可能错误的假设。找出这件事可能失败的所有方式。尽可能强硬地为相反观点辩护。不要客气。不要加限定条件。只管攻击。之后，为我的立场做最强的论证。建构一个你能构建的最有力的理由来说明为什么我是对的。然后告诉我你真实的想法。
08 扩展思考模式

大多数 Claude 用户从来没打开过这个功能。Extended Thinking 是让 Claude 在给出答案之前，一步一步地推理问题，而不是直接出结果。

简单任务不需要用它。复杂决策、分析、或者你希望 Claude 真正思考而不是模式匹配的问题，打开它。

在 Claude 里，发送消息前点击脑子图标。或者在 prompt 里加这段：

Prompt 
仔细思考这个问题再回复。一步一步推演，展示你的推理过程，标出你不确定的地方，然后给我你的结论。

Prompt

仔细思考这个问题再回复。一步一步推演，展示你的推理过程，标出你不确定的地方，然后给我你的结论。

在难题上，输出质量的差异是明显的。

09 让 Claude 给 Claude 写 Prompt

这是最被低估的用法。如果你不知道该怎么给某个任务写 prompt，直接问 Claude 帮你写。

Claude 知道什么样的指令能产生更好的结果。让它把这个能力用在你身上。

Prompt 
我需要 Claude 帮我 [描述你的实际任务]。

为我写一个这个任务最好的 prompt。包含角色、上下文、格式指令和任何能提升输出质量的约束条件。

然后立刻用这个 prompt 帮我完成任务。

Prompt

我需要 Claude 帮我 [描述你的实际任务]。为我写一个这个任务最好的 prompt。包含角色、上下文、格式指令和任何能提升输出质量的约束条件。然后立刻用这个 prompt 帮我完成任务。

PART 04

少花 Token，多拿价值

10 精确指定输出长度

Claude 默认会写它觉得合适长度的内容。通常比你需要的更多——意味着更多 token、更长阅读时间、更多噪音。

在它开始之前，告诉它你想要多长。

Prompt 
答案最多 3 句话。
Prompt 
给我 5 个要点。不需要解释，只要要点。
Prompt 
写这篇文章，字数控制在 150 字以内。

Prompt · 示例

答案最多 3 句话。给我 5 个要点。不需要解释，只要要点。写这篇文章，字数控制在 150 字以内。

这一句话 instruction，大多数任务可以减少 40% 到 60% 的 token 消耗，而且你需要的价值一点没少。

11 去掉废话开篇

Claude 默认每次回答开头都会加一段你没要的东西。"好问题，让我来帮你分析一下"、把你刚说的话复述一遍、加一段免责声明、结尾再来个总结重复刚说的内容。这些你都没要求，它每次都加，消耗 token，浪费你时间。

把这段加进你的 Custom Instructions：

Prompt 
永远不要用开场白、肯定语或复述我的问题作为回复开头。直接给答案。除非我明确要求，否则不要在结尾加总结。除非话题确实需要，否则不要加免责声明。

Prompt

永远不要用开场白、肯定语或复述我的问题作为回复开头。直接给答案。除非我明确要求，否则不要在结尾加总结。除非话题确实需要，否则不要加免责声明。
12 不要每次对话都重新解释自己

如果你在每个新 Chat 里都在粘贴同样的背景信息，你每轮对话都在浪费 token，而且还在培养一个随着 Claude 使用量增加成本越来越高的习惯。

这就是 Projects 和 Custom Instructions 存在的意义。背景信息存一次，Claude 每次对话开始自动读取。再也不用粘贴了。

还没开始用 Projects？先把这件事做了，比这篇文章里任何其他技巧都重要。

13 换话题就开新 Chat

Claude 会携带当前对话中所有说过的上下文。当你在一个很长的 Chat 里换话题，Claude 仍然加载着之前所有的上下文。这意味着每次回复 token 消耗更多、处理更慢，上下文污染让你的新话题受到之前内容的影响。

当切换到不相关的话题，在 Project 内开一个全新的 Chat。Project 记忆保留，不相关的包袱丢掉。

PART 05

直接拿去用的 Prompt 模板

14 用费曼法理解任何东西

Claude 默认给出的解释技术上都正确但实践中用处不大。它们使用了和要理解的东西一样的词汇，意味着你带走了一个定义但没有真正的理解。

费曼法通过简单性强迫真正的理解。如果 Claude 不能用日常类比简单解释清楚一个东西，说明这个解释还不够清楚。这个 prompt 适用于任何东西——投资学、量子物理、某个 API 是怎么工作的。

Prompt 
用类比和日常例子解释 [主题]。不要用术语。假设我在这个领域没有任何背景知识。

每个类比之后，通过问我一个问题来确认我是否真正理解了。根据我的回答，加深或调整解释。

一直继续，直到我能用自己的话解释它而不使用任何技术术语。

Prompt

用类比和日常例子解释 [主题]。不要用术语。假设我在这个领域没有任何背景知识。每个类比之后，通过问我一个问题来确认我是否真正理解了。根据我的回答，加深或调整解释。一直继续，直到我能用自己的话解释它而不使用任何技术术语。
15 按你真实旅行风格做的旅行规划

大多数旅行规划从目的地开始，给你一个去任何旅行博客都能找到的通用行程。Claude 可以做一件不同的事：按你具体的旅行风格、节奏、预算、真正在乎的东西来规划，而不是什么热门打卡清单。

关键是给它真实的信息，不只是日期和地点。

Prompt 
我计划去 [目的地]。会在那里待 [天数]。预算大约是 [金额] 每天，包含住宿。

这是我的旅行风格：[描述你的风格——你喜欢睡到自然醒还是行程紧凑、热门景点还是本地人去的地方、博物馆还是美食、活跃还是休闲、一个人还是结伴，等等]。

我想避免的：[人群、网红打卡地、天价餐厅、长途交通等]。

按这个风格为我制定每日行程。对每一天，包含住宿地点、做什么、吃什么、以及需要知道的任何后勤信息。标注任何需要提前预订的项目。

我计划去 [目的地]。会在那里待 [天数]。预算大约是 [金额] 每天，包含住宿。这是我的旅行风格：[描述你的风格]。我想避免的：[人群、网红打卡地、天价餐厅、长途交通等]。按这个风格为我制定每日行程。对每一天，包含住宿地点、做什么、吃什么、以及需要知道的任何后勤信息。标注任何需要提前预订的项目。
16 月度开支分析，得出真实结论

大多数人在看银行账单时，只是模模糊糊地感觉花了太多，但没有真正搞清楚到底发生了什么。Claude 可以把原始数字变成一张清晰的图：钱花在了哪里，你应该怎么做。

粘贴真实数据效果最好，而不是估算。

Prompt 
我会粘贴上个月的支出。帮我分析并告诉我：

1. 我在哪些类别上花费最多
2. 和我说的目标相比，我的支出哪里不正常
3. 什么可以砍掉但不影响生活质量
4. 我可能在哪些重要地方花得不够
5. 一个最有财务影响力的具体改变

这是我的支出：[粘贴银行账单或支出列表]

我现在的财务目标：[描述你想做什么——存更多钱、还债、投资等]

Prompt

我会粘贴上个月的支出。帮我分析：我在哪些类别上花费最多，和目标的差距，哪里可以砍掉不影响生活，哪里可能花得不够，一个最有影响力的改变。[粘贴支出数据] 我现在的财务目标：[描述目标]
17 你的个人思考伙伴

大多数人的生活中没有一个愿意倾听、不带评判、问对问题、帮你把困惑想清楚、同时不会把自己的议程强加给你的人。Claude 可以填补这个角色，但前提是你给它正确的指令。

这不是治疗。这是结构化的自我反思，加上一个能帮助你更清晰思考的外部视角。

Prompt 
我想和你聊一聊我正在纠结的一件事。你的任务不是立刻给我建议。

首先，问我问题，全面了解情况。实际发生了什么，我对此有什么感觉，我试过什么，我希望什么结果。

等你全面了解之后，反馈你听到的内容——不只是事实，还有表面之下似乎隐藏的东西。

然后，也只有到那时，再给我你的观点。诚实，而不是让我舒服。告诉我你真实的想法，包括我可能不想听的内容。

我心里在想的是：[描述你想思考的事情]

18 在投入之前压力测试任何商业想法

大多数商业想法死掉，是因为人在测试之前就已经爱上了它们。他们花好几个月做出一个没人要的东西，因为从来没有诚实地问过这个想法是不是真的好。

Claude 可以充当一个无情的第一道过滤器。不是为了否定想法，而是在花时间和金钱之前找到真正的问题。

Prompt 
我有一个商业想法，想在投入大量时间之前压力测试一下。

想法是这样的：[详细描述——是什么，给谁用，怎么赚钱，为什么你认为它可行]

你的任务是找出它所有的问题。具体来说：

1. 我做了哪些可能错误的假设
2. 谁已经在做这个，我为什么可能输给他们
3. 目标客户为什么可能实际上不会为此付钱
4. 这件事要成功必须为什么条件成立，可能性有多大
5. 这个想法最大的一个问题是什么

要具体。"市场可能还没准备好"这种泛泛的风险没用。给我每个问题的真实版本。
