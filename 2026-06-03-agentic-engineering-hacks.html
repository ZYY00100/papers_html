Agentic Engineering · 实战工作流

Every Agentic Engineering Hack

从 vibe coding 到一个人就是一家公司

by @朱有以 Matt Van Horn @mvanhorn 2026-06-03

Lyft 联创 3 个月前那条 91.3 万阅读的"Every Claude Code Hack I Know"重写了。这次他把所有 hack 升级成 Agentic Engineering 22 条，配 22 段原文 + 22 句中文注释。

前言

Matt Van Horn，Lyft 联合创始人。2026 年 3 月在 X 上发了 "Every Claude Code Hack I Know"，阅读量 91.3 万。3 个月后他升级了版本：22 个 hack，全是实战工作流，不是工具介绍。

他自己交代了为什么要写：

I hadn't shipped software anyone valued since high school. These are my hacks. —— @mvanhorn

意思是他从 Lyft 出来之后"沉默了十多年"，今年才因为 agentic engineering 重新开始 ship——并陆续把 Python、Go、Compound Engineering、Superpowers 等多个顶级开源项目推到贡献榜前列。原文末尾的"真实 afternoon"那一段最值得读：让 agent 暖车、买 Costco 啤酒、盯比赛、订机票——一个人从工地、车上、咖啡馆全在 ship。

这一篇的核心是搬运 + 注释。22 个 hack 一个不漏，每条配中文场景注释，让你读完知道"我能不能在自己的活儿里套这个"。

Agentic Engineering 是什么

Matt 自己给了一个简洁的版本：

This used to be called vibe coding. Around last Thanksgiving the models got good enough that the toy became real, what people now call Agentic Engineering. It's the only reason I ship. —— @mvanhorn

翻译一下：vibe coding 一直是"用嘴写代码"，但去年感恩节前后模型能力过了临界点，玩具变成了真工具——这个新阶段叫 Agentic Engineering。区别在哪？

维度Vibe CodingAgentic Engineering
核心动作 用自然语言驱动 AI 写代码原型 把整套工程流程（plan / build / verify / ship）外包给 agent
人的角色 提示词写手 + 调试员 "human signal"——决定方向、做判断、定标准
规划占比 边写边想 80% 规划 + 20% 执行（彻底翻转）
并行能力 单线任务 4-6 个 cmux tab 同时跑多个 session
交付物 一次性代码片段 可复用 skill + 持续 ship 的真项目

从 Matt 自己的成果看：3 个月前他只能写 91.3 万阅读的"hack 列表"，3 个月后他把 last30days 推到 27K 星，Printing Press 4K 星，自己成为 5 个顶级项目的 Top 3 贡献者。这条赛道的人是真在"用 AI 重新变成高产开发者"。

黑话词典（读之前先扫一眼）

这 22 个 hack 里塞了一堆名字工具，不熟悉的先看这里：

·

Compound Engineering（CE）

Every 公司出的 Claude Code 插件。核心是两个斜杠命令：/ce-plan 自动产出 plan.md，/ce-work 按 plan 执行。Matt 是这个项目第 3 贡献者。

·

last30days

Matt 自己写的 AI 调研 skill。并行搜 Reddit / X / YouTube / TikTok / HN / GitHub / Polymarket / 通用网页，按真实互动量排序后由 AI 评委综合成简报。27K 星。

·

Printing Press

Matt + trevin 做的"CLI 工厂"——给真实世界服务（Tesla、Instacart、ESPN、Alaska 航空……）造 agent-native 命令行界面，让 agent 能像人一样去操作它们。4K 星。

·

Agent Cookie

Printing Press 的最新发布。把你的真实浏览器 cookie 喂给 CLI，让 agent 以"你"的身份去操作网站，不用再贴密码重新登录。

·

cmux

支持多 tab 状态保存的终端窗口管理工具。Matt 的多 session 并行工作流就是基于它。Mac only。

·

Ghostty

高性能终端模拟器，开源。Matt 以前主力，现在用 cmux。

·

Monologue / Wispr Flow

Mac 上的语音转文字工具。直接说话、上下文由 LLM 补全那种。

·

Granola

会议记录 AI 工具，把对话转成结构化笔记。Matt 用它攒"raw transcript"喂给 agent。

·

HyperFrames

把视频当作 HTML 写的工具——agent 能直接产出 MP4。Matt 用它做产品 demo 视频。

·

Proof

把 markdown / spec 转成可分享带评论的网页，让非技术同事能在 plan.md 上批注，批注回流 agent 循环。

·

AgentMail

给 AI agent 用的电子邮箱。给一个 agent 一个邮箱地址，发邮件就能开新 session。

·

Mosh / Tmux / Hermes / OpenClaw

远程工作栈：Mosh 在弱网下维持 SSH 体验；Tmux 在飞行模式下挂载会话；Hermes 和 OpenClaw 是 Matt 同时跑的两个 agent 框架，前者擅长自学习循环、后者擅长广度。

·

Codex

OpenAI 的 coding agent。Matt 把它当 Claude 的"第二引擎"——Claude 负责 plan 和 taste，Codex 负责大并行 build。两边都开 xhigh 推理。

术语扫完了，下面上 22 个 hack。

22 个 hack · 逐条搬运

每条格式：原文（Matt 自己的话）+ 一句"它解决什么"+ 一句"你的场景"。

00 YOLO TL;DR Hack（开局这一条最野）
paste this entire article to your agent and tell it to make a plan to set up everything in it, then work that plan one hack at a time. That's my whole stack, no reading required.

它解决什么：懒人启动器。不用读完 22 个 hack，直接把整篇文扔给你的 agent，让它给你出 plan，再一项项执行。

你的场景：读完这篇文，最后一步就是这一招——把这整篇 HTML 复制粘贴到 Claude Code 里。

全文 CTA
01 想到 idea 的第一秒就 /ce-plan
The moment I have an idea, it's /ce-plan to make a plan.md. Not "let me think about this," not "let me start coding." /ce-plan, every time.

它解决什么：默认动作改成 plan 而不是 code。截图、报错、issue URL、设计稿、Slack 讨论都能直接喂进去。

你的场景：把"想到就打开 IDE"换成"想到就 /ce-plan"。哪怕 idea 模糊，先用 /ce-brainstorm 把它聊清楚，再 /ce-plan。

核心 · 必装
02 plan.md 是给 agent 看的，不是给你
I always make the plan.md. I almost never read it. Plans are for agents, you silly human.

它解决什么：反"agent 偷懒"。有 plan 的 agent 会真正做完整工作；没 plan 的 agent 走捷径半途而废。plan.md 是一根"皮绳"。

你的场景：运行 /ce-plan 之后只扫一眼标题，不读 300 行细节。卡住时用 "TLDR?" / "eli5 this plan" / "wait, why this approach?" 在 session 内追问。

反 AI 偷懒
03 用 /ce-plan 做最深的非工程工作
The biggest thing I've learned since March is that they aren't. The deepest knowledge work I do now runs through the same loop, and the trick is to make the first plan a plan for the plan.

它解决什么：让 LLM 不偷懒的硬技巧——直接问"产出"它会偷懒；先问"产出的计划"再问"执行"它会做完整。套到任何深度思考场景：商业战略、PRD、竞品分析、董事会汇报。

你的场景：下次要写一份重文档，先别让 agent 直接写。先给它"原材料 + 目标描述 + '先做 plan for the plan'"，等 45 分钟拿到一个能用的 plan，再 /ce-work 跑出真稿。

深度思考
04 上语音
Voice-to-LLM is different from voice-to-anything-else. The transcription doesn't have to be perfect because the listener understands context.

它解决什么：打字的瓶颈比想的严重。LLM 听半句也能猜出你意思——你嘴里嘟囔一半、它补全另一半。

你的场景：Mac 装 Monologue 或 Wispr Flow，配个鹅颈麦；手机直接用 Apple 听写。Matt 自己承认在开放工位还没解决"不出声"的问题，如果你解决了，告诉他。

效率倍增
05 cmux 开 4-6 个 tab
Four to six cmux tabs, sometimes more, each a separate session: one writing a plan. one building from a different plan. one running last30days. one fixing a bug I found testing the last thing.

它解决什么：把"agent 在跑"这件事做成多窗口并行。一个 /ce-plan 调研的功夫，切到另一个 /ce-work 已经写好的 plan。等它跑完，切去第三个修刚发现的 bug。

你的场景：从"我有一个 session"升级到"我有 4-6 个同时跑的 session"。关键基建：cmux + 声音 hook（哪个 tab 跑完会响）。

并行
06 新 tab 直接进 Claude，不要进 shell
A new tab should open straight into Claude Code, not a shell. Open a tab, you're already talking to an agent. No cd, no typing claude.

它解决什么：降低"开新 session"的成本。一个键就能开一个 session 时，你会开得更多。

你的场景：Ghostty / cmux 配置里加一行 command = ~/.local/bin/claude-launcher.sh，每次开 tab 直接进 Claude。完全不用 cd 项目目录——agent 自己找。

基建
07 远程控制 + 给 Claude 一个邮箱
Turn on remote control every time you open a new window. Give your Claude an email address with AgentMail. Email the inbox and a fresh session opens and starts working on whatever's in the subject and body.

它解决什么：让你的 Mac 在家跑着、人在外面能直接戳——手机上发个邮件就开新 session。open source 在 github.com/mvanhorn/agentmail-to-claude-code。

你的场景：把 remoteControlAtStartup: true 写到 ~/.claude/settings.json；装 AgentMail 邮箱，allowlist 只放你自己的地址，SPF/DKIM 失败一律丢。

远程 · 基建
08 危险模式 + 声音 hook
With six sessions you can't babysit it. skipDangerousModePermissionPrompt: true is the key. The other setting is a sound hook, non-negotiable with six sessions.

它解决什么：同时跑 6 个 session 时，每条确认都会打断你。"dangerously skip permissions"是必选项，配套一个 Stop hook 跑完时响一声——人靠声音定位"哪个 tab 跑完了"。

你的场景：把那段 ~/.claude/settings.json 复制过去。Codex 同理：approval_policy = "never" + sandbox_mode = "danger-full-access"，或启动时加 --yolo。GitHub 是兜底，崩了能恢复。

YOLO
09 Codex 不开 CLI，三种姿势交给它
I send work to Codex all day, and I almost never open the Codex CLI to do it. Claude plans, Codex builds, and I never leave my Claude session.

它解决什么：两个 $200 plan 并排用 = 第二个引擎。三种交付方式：Codex IDE 扩展（不离开 Claude）、/ce-work --codex（CE 内部委托）、Printing Press 末尾加 codex（CLI 生成时直接交付给 Codex）。

你的场景：Codex 开 reasoning xhigh + fast mode；Claude 开 xhigh + 关闭 fast mode（fast 会额外按 token 计费）。大并发 build 走 Codex，planning 和 taste 留给 Claude。

多模型协作
10 plan 之前先 /last30days
Before I /ce-plan, I usually run /last30days on it first.

它解决什么：用实时社区共识代替 6 个月前的训练数据。挑库、写 feature、见客户、写文章之前都跑一遍——相当于给自己一份"最近一个月大家怎么看这件事"的简报。

你的场景：选 Vercel agent-browser 还是 Playwright？"/last30days Vercel agent browser vs Playwright"，几分钟拿到真实对比。

调研
11 把 raw 会议记录直接喂给 LLM
The trick is raw. I don't summarize first. I drop the whole messy transcript in, tangents about sushi and all, and let Claude do the extraction.

它解决什么：Granola 的转录 + 你的 codebase + 你的历史 strategy plan，三件套丢给 agent，比人脑回忆清楚得多。

你的场景：客户拜访完、面试完、和老板 1:1 完，别自己写小结。整段 transcript 粘进去，/ce-plan turn this into a product proposal。Printing Press 的 Granola CLI 能把"调出任意历史会议"做成一行命令。

会议 × Agent
12 做"人味信号"而不是"动手的人"
When you run six agents, your job is not to do the work. Your job is to be the signal. The agents supply volume. You supply taste, direction, and the react-and-redirect loop.

它解决什么：心态转换。6 个 agent 在跑时，最贵的不是你的手速，是你的判断——"option 2 方向对但用 option 1 的语言"、"这个风险点先解决"、"这段太长"。

你的场景：从"自己写"切到"看 agent 写 → 指挥修改"。越快切换到 taste 和 direction，ship 越快。

思维范式
13 用 HyperFrames 做视频，跟写代码一样
HyperFrames lets me build video as HTML, so an agent can write it. The loop is identical to code, the output is just an MP4 instead of a PR.

它解决什么：视频制作门槛被抹平——每个 product demo、launch reel、技术讲解都能有视频版本。脚本用 script.md 写、agent 渲染、产出 MP4。还能直接嵌进 PR、README。

你的场景：把"要不要做视频"换成"要做哪种视频"。上传 GIF 到 catbox 在 GitHub 上能直接渲染。

视频
14 你的笔记 = agent 的知识库
The reason a plan gets better every time is that Claude has access to every prior plan I've written. Compounding context.

它解决什么：个人 RAG。Bear 笔记、Obsidian vault、gbrain 跨设备同步、supermemory agent memory layer——选一个有 CLI 或 API 的，把 agent 指向它，越用越聪明。

你的场景：选你已经在用的笔记工具，给 agent 配 CLI。10 年的笔记 + 全部历史 plan = 你的私人 RAG。compound context，越用越强。

个人 RAG
15 Mac mini 远程栈
Mosh when you have to SSH in. It keeps the session local-feeling and responsive over bad wifi and roaming. On plain SSH, Claude Code crawls, every keystroke waiting on the round trip.

它解决什么：远程工作栈：Mosh（弱网 SSH）+ Tmux（飞机模式会话保活）+ Hermes / OpenClaw（远端自主 agent）+ Agent Cookie（跨机 cookie / .env 同步）。

你的场景：如果你常在路上写代码，这套栈能让"飞行 8 小时还在 ship"成为可能。Matt 飞欧洲回家那一程就在 ship。

远程
16 用 Proof 把 plan 发给非技术同事
A plan.md is perfect for me and useless to hand to someone who doesn't live in a terminal. That was the last real gap, and Proof closed it.

它解决什么：plan.md 给自己看是神器，给产品经理看是梦魇。Proof 把 md 转成可分享的网页，评论回流 agent 循环。这才是"人在循环"真正落地。

你的场景：下次有跨职能评审，把 plan.md 丢 Proof，发链接，对方在网页里评论，你把评论粘回 agent 继续改。

协作
17 写自己的 skill
Anything I do more than twice, I turn into a skill: a reusable command my agents can run forever.

它解决什么：从"用 agent"升级到"教 agent"。你做超过 2 次的事，就值得固化成一个 skill，下次所有 session 都能复用。Compound Engineering 的"compound"就是这个意思。

你的场景：跟 agent 说："看看 Compound Engineering 的 skill 怎么写的，帮我照着做一个 [X 领域] 的"。agent 会读一个好范例、学会结构、scaffold 出你的。last30days 起源就是这么做的。

复利
18 给用得最多的开源项目提 PR
Not drive-by typo fixes, real features on tools I use every day. Somewhere along the way I started landing near the top of the contributor lists: #3 on Compound Engineering, Superpowers, and Emdash.

它解决什么：同一套 /ce-plan + /ce-work 流程既能 ship 自己的项目，也能 ship 别人的项目。Matt 用这套进了 Python、Go、OpenCV、Superpowers 等项目的 Top 10 贡献者。

你的场景：挑你每天用的工具，找到一个真缺的小功能，用同样循环 PR 进去。PR 是门票，Discord 里认识的人才是回报（Matt 用这招招到一个工程师）。

开源 + 招人
19 M5 Max + Anker 充电宝 + 车里充电器
My two-year-old laptop was barely functional under everything I run on it, six Claude sessions plus Codex all day. So I upgraded to an M5 Max with 64GB of RAM.

它解决什么：硬件真顶不住——6 个 Claude + 1 个 Codex 同时跑，新机 M5 Max 64GB 续航也撑不过 1 小时。

你的场景：至少 64GB 内存（否则 swap 拖累），SSD 越大越好。Anker 充电宝随身、Tesla 里塞充电器、sudo pmset -a disablesleep 1 别让电脑睡着。

硬件
20 Printing Press：让 agent 跑你的生活
Printing Press is a fleet of CLIs that wrap real-world services so an agent can just do the errand. Tesla preheat. Instacart. ESPN polling. Alaska Airlines.

它解决什么：agent 不仅能写代码，还能帮你暖车、买菜、盯比赛、订机票。Agent Cookie 是最近的拼图——把你的真实浏览器会话交给 CLI，让它以"你"的身份操作网站。

你的场景：在 printingpress.dev 找现成的 CLI 用。最强的玩法是：自己 day-to-day 用的服务，让 Printing Press 给你生成一个 agent-native CLI。你自己天天用的那个，就是改变你工作方式的那个。

errand · CLI
21 诚实的一节：AI psychosis
Agents were supposed to do all the work for us. Instead, every friend I have is working the hardest they ever have in their lives. Building with agents is the greatest video game ever made, and the loop is that good.

它解决什么：这是全文唯一不带工具的 hack，是警告。Matt 说他有朋友"build 到停不下来，launch 了没人用，自己也不在乎——因为 build 本身就是奖励"。

你的场景：照照镜子。问问自己：这个东西真的有人想要吗？如果答案只是"我自己用"——那也没问题，但别让它吞掉你周围的人。休息、跟身边的人说话、touch grass。

反思
22 这篇文章是这样写出来的
Claude Code in cmux, me talking into Monologue: "evolve the no-IDE opener," "make the don't-read-the-plan section spicier," "add the Tesla and Instacart story." It rewrites, I react, and it's in Proof for review.

它解决什么：这就是 22 个 hack 自身的 meta-demo——他用 voice + cmux + Proof + last30days 写了这篇文。talk → plan → build → review。桌椅上、车里、足球场都能写。

你的场景：你也可以这样写——把整篇文粘贴给你的 agent，让它自己出 plan，再一项项执行。YOLO TL;DR 玩法。

meta

最后说两句

22 个 hack 看下来，你会发现它们不是 22 个独立技巧，是 1 套连贯的方法论：

  • Plan first, never read the plan. 让 agent 做计划、agent 执行、你只做判断。
  • Compound everything. 每次学到的 trick 写成 skill，下次所有 session 复用。
  • Be the human signal. 你给方向和 taste，agent 给速度和广度。
  • Run multiple in parallel. 4-6 个 tab、多个模型、多个 cmux 窗口，声音 hook 定位哪个跑完了。
  • Be careful. AI psychosis 是真的风险。第 21 条是警告不是装饰。

Matt 给了一个特别值得记住的对照——vibe coding 是"用嘴写代码"，agentic engineering 是"把整套工程外包给 agent"。两者在 prompt 层面一样，区别是后者要 plan、要并行、要复利、要自建 skill。

上一篇文章里 Thariq 把 dynamic workflow 摆出来——是 Anthropic 给"agent 时代"的脚手架。这篇 Matt 的 22 个 hack 是脚手架上面跑的人的真实生活。两条线一对照，2026 年 agent 开发的图景就基本完整了。
