# Claude Code 的 Agent View 功能：让多个 AI 同时帮你打工

今天刷 X 的时候，看到 Claude 官方发了一条新功能推送——Agent View，配了一个界面截图，看完第一反应是"这不就是多任务管理吗"。

但点进去官方文档读了几分钟，发现不是我想的那样。这个功能解决的不只是"同时开几个窗口"的问题，而是重新定义了一个人和 AI 的协作方式。

这篇文章就是我读完官方文档后的整理，想写给和我一样对 AI 工具感兴趣、但不想被技术细节绕进去的人。

---

你有没有过这种感觉——手上的活堆成山，改代码、写文档、看 PR，每一个都急，但 AI 只能一个一个来，排队排到崩溃。

上周 Claude Code 更新了一个功能，叫 **Agent View**，解决的就是这个问题。它能让你同时调度多个 AI 干活，每个各干各的，互不干扰。你只需要坐在监工的位置上，盯着就行。

---

## 多 agent 协作：不是 AI 变强了，而是多了几个帮手

在说 Agent View 之前，得先理解它背后的概念——多 agent 协作。

你以前用 AI，可能是一个问题抛出去，AI 回复了，你再问下一个。它再强，也是一次只能干一件事。

多 agent 协作的意思是：**你同时派出去好几个 AI，各自从不同的方向帮你干活。**

举一个具体的场景——

你今天要同时做三件事：
- 修一个 bug
- 写一份设计文档
- 回复 Code Review 意见

以前你只能排优先级，AI 干完一件你再喂下一件。现在你可以同时开三个 session，一个修 bug、一个写文档、一个看 PR，**三个 AI 同步在线，各干各的**。

这就像你同时雇了三个实习生，每个人独立负责一摊。你是老板，分配任务、看进展、在关键节点把关。

不是 AI 变聪明了，是**干活的 AI 变多了**。

---

## Agent View 是什么

那怎么同时管这么多 AI？

Claude Code 给了一个专门的界面，叫 **Agent View**。

你只需要在终端里运行：

```bash
claude agents
```

就会看到这样一个界面——

（这里会展示 Agent View 的截图）

这张表里，每一行就是一个正在跑的任务。你一眼就能看到：
- 哪个在干活（Working）
- 哪个卡住了，需要你回复（Needs Input）
- 哪个已经完成了（Completed）

不用再去翻一个个聊天记录，直接在这里一览无余。

---

## 怎么用

### 打开界面

在终端输入 `claude agents`，回车，界面就出来了。最底部有一个输入框。

### 分发任务

在底部输入你想让 AI 干的事，回车，它就启动一个新的独立 session 开始跑。你可以同时输入多个任务，每个都是独立并行跑起来的。

### 查看进展

你想知道某个任务在做什么？选中那一行，按**空格**，会弹出一个 peek 面板，显示它最近在做什么、输出了什么。不用离开 Agent View，不用打断其他任务。

### 回复和操作

在 peek 面板里，你可以直接打字回复它。比如它问你要一个决策，你直接在面板里回答，回车发送，整个过程不用离开这个界面。

### 深入操作

如果你觉得某个任务需要更深度地介入，选中那行，按**回车**或者**右箭头**，就"钻进"了那个 session，像正常使用 Claude Code 一样去操作。操作完，按**左箭头**回到 Agent View。

### 整理界面

- 按 `Ctrl+T` 把某个任务钉到顶部，方便关注
- 按 `Ctrl+S` 在"按状态分组"和"按目录分组"之间切换
- 按 `Ctrl+R` 给任务改名，方便识别

---

## 几个说明

**关于隔离**：每个 session 都在独立的 worktree 里运行，不会互相干扰文件编辑。你改代码的时候不会影响到另一个 session 在写的文档。

**关于状态**：session 有六种状态——Working（运行中）、Needs Input（等你回复）、Idle（闲置）、Completed（完成）、Failed（出错）、Stopped（手动停止）。如果某个任务没响应了，看看状态栏。

**关于持续运行**：只要你的电脑不关机睡眠，这些 background session 会一直跑。你关掉 Agent View 也没事，下次打开 `claude agents`，它们还会在那里等你。

---

## 为什么这个更新值得关注

Agent View 不只是一个"多任务管理界面"。它代表了一种工作思路的变化：**从"问 AI 一个问题"到"调配一群 AI"**。

以前你是单兵作战，AI 是你的辅助工具。现在你可以把 AI 当成团队来用，每个人各司其职，你做资源调配和决策把关。

这种工作流特别适合需要并行处理、周期较长的任务——技术调研、代码审查、文档撰写、内容创作，都可以用这种方式来加速。

如果你的电脑上跑着 Claude Code，现在就可以试试 `claude agents` 这个命令，打开看看。也许你会发现，原来一个人也能同时推进好多事情。
